# Design System Guidance

This file is the design system source of truth for agents and builders.

## Principles

- Prefer existing components before creating new ones.
- Use design tokens for color, spacing, typography, and elevation.
- Keep hierarchy clear.
- Make primary actions obvious.
- Avoid adding unrequested actions.
- Preserve accessibility.
- Match nearby product surfaces unless the Change intentionally creates a new pattern.

## Agent guidance

Before changing UI:

1. Inspect nearby screens.
2. Identify existing components.
3. Use tokens rather than raw values.
4. Capture before/after screenshots.
5. Run accessibility and browser checks.
6. Do not add new product behavior unless requested.

## Repeated findings

Promote repeated design review findings into this file, component examples, tests, or lints.
