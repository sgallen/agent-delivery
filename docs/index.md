# Playbook Map

This playbook has four layers. Read only as deeply as the problem requires.

## 1. Understand the shift

Start with [Why This Exists](00-the-case/00-why-this-exists.md), then read [The Case](00-the-case/index.md).

It explains why current agents are more than a productivity feature, why builder attention and other surrounding capacities become the bottleneck, and why the operating model needs to change.

Recommended short path:

1. [Why This Exists](00-the-case/00-why-this-exists.md)
2. [The Economics Changed](00-the-case/01-the-economics-changed.md)
3. [The Cost Curve Made Concrete](00-the-case/02-the-cost-curve-made-concrete.md)
4. [The Bottleneck Moved](00-the-case/05-the-bottleneck-moved.md)
5. [Invert the Default](00-the-case/06-invert-the-default.md)
6. [The Purpose Is Outcomes](00-the-case/08-the-purpose-is-outcomes.md)
7. [The North Star](00-the-case/10-the-north-star.md)

## 2. Understand the system

Read the [Operating Model](01-operating-model/index.md).

Start with:

1. [Changes](01-operating-model/changes.md)
2. [Not All Work Is Merge-Bound](01-operating-model/not-all-work-is-merge-bound.md)
3. [`WORKFLOW.md` and Workpads](01-operating-model/workflow-and-workpads.md)
4. [Gates](01-operating-model/gates.md)
5. [Evidence](01-operating-model/evidence.md)
6. [Resource Observability and Delivery Economics](01-operating-model/resource-observability-and-delivery-economics.md)
7. [Initiatives and the Value Loop](01-operating-model/initiatives-and-value.md)
8. [Builder Engagement](01-operating-model/builder-engagement.md)

The first five are enough for a small experiment. Resource observability begins with lightweight attribution. Initiative and capacity practices are for work whose investment and value deserve a durable record.

## 3. Try it

Use the [First Experiment](02-playbooks/first-experiment.md) to run one low-risk Change through the model. For a concrete bug workflow, use [Fix a Bug with Proof](02-playbooks/fix-a-bug-with-proof.md).

For a team-level start, use [The First 30 Days](02-playbooks/first-30-days.md), then choose a path from the [Playbooks](02-playbooks/index.md): greenfield, existing-team adoption, GitHub workflow, gates, browser proof, environments, delivery economics, initiative forecasting, capacity planning, non-landed resolution, or recovery.

## 4. Implement it

Use the [Reference](03-reference/index.md) and `templates/` directory for copyable starting points.

The templates demonstrate one GitHub-first implementation, including run-resource and initiative records. Keep the operating principles; adapt the tools and data retention to the project.
