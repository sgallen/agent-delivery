# Reference: Pull Request Template

A pull request is a review package. It is not the lifecycle.

The starter is [`templates/.github/PULL_REQUEST_TEMPLATE.md`](../../templates/.github/PULL_REQUEST_TEMPLATE.md).

## What the PR should make easy to judge

- what became observably true;
- the criteria, gates, and evidence;
- scope, risk, rollout, rollback, and continuing obligations;
- known gaps or waivers;
- the learning or follow-up.

When the branch carries experiment or investigation evidence rather than a merge candidate, make the decision question and conclusion explicit.

Resource actuals, initiative updates, and formal resolution fields belong in the PR only when the repository uses them.

A merged PR may still be unreleased. A closed PR may support a strong decision not to continue. The PR should help the builder judge the work without pretending that its existence determines the outcome.
