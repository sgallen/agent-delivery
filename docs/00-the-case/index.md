# The Case

Traditional product delivery is not broken. It has produced enormous value, and it still works.

It was also built around a constraint that has changed: meaningful implementation once required scarce human capacity. Agents can now participate directly in that work.

That does not make product judgment, engineering discipline, design taste, security, or accountability less important. It changes where those scarce human capabilities create the most leverage.

The argument unfolds in this order:

1. [**Why This Exists**](00-why-this-exists.md) — the experience behind the playbook and the gap between better tools and an unchanged system.
2. [**The Economics Changed**](01-the-economics-changed.md) — implementation capacity now has a different marginal cost and can be observed more directly.
3. [**The Cost Curve Made Concrete**](02-the-cost-curve-made-concrete.md) — a dated price snapshot, with the caveats intact.
4. [**Current Capability Is Enough**](03-current-capability-is-enough.md) — the case does not depend on future model magic.
5. [**The Advantage Compounds**](04-the-advantage-compounds.md) — teams improve their operating system while the tools may improve too.
6. [**The Bottleneck Moved**](05-the-bottleneck-moved.md) — builder attention, shaping, review, infrastructure, and release capacity can matter more than code output.
7. [**Invert the Default**](06-invert-the-default.md) — design around agent execution, then place human judgment deliberately.
8. [**The Old Process Is Not Enough**](07-the-old-process-is-not-enough.md) — not because it was foolish, but because it was built for different constraints.
9. [**The Purpose Is Outcomes**](08-the-purpose-is-outcomes.md) — delivery activity only matters when it improves the product, the decision, or the system.
10. [**Builders, Not Silos**](09-builders-not-silos.md) — specialties remain; handoffs should not define the work.
11. [**The North Star**](10-the-north-star.md) — human-governed, agent-enabled product delivery.
12. [**Company-Building Implications**](11-company-building-implications.md) — different constraints produce different teams, capital needs, and investment choices.
13. [**Beyond Product Development**](12-beyond-product-development.md) — product delivery is the first proving ground, not the boundary.

## Keep the hard parts in view

Capable agents inside a weak operating model create more material for humans to inspect and repair. That is not leverage. It is a faster way to move the bottleneck.

The playbook therefore treats intent, environments, gates, evidence, resource use, and builder judgment as part of the delivery system. It also keeps a firm distinction between a useful decision not to land and work that simply disappeared.

Do not lower the bar. Build a system that can clear it more reliably.
