# The Case

Traditional product delivery is not broken. It has produced enormous value, and it still works.

The problem is that it was designed around a constraint that has changed: meaningful implementation work once required scarce human capacity. Agents can now participate directly in that work.

That does not eliminate the need for product judgment, engineering discipline, design taste, security, or accountability. It changes where those scarce human capabilities create the most leverage.

This section begins with the experience that produced the playbook, then builds the argument in sequence.

## The argument

1. [**Why this exists**](00-why-this-exists.md) — direct use, team adoption, and the gap between better tools and an unchanged operating model.
2. [**The economics changed**](01-the-economics-changed.md) — implementation capacity is no longer scarce in the same way, and delivery can become more observable and learnable.
3. [**The cost curve made concrete**](02-the-cost-curve-made-concrete.md) — a dated price snapshot makes the new cost envelope tangible without pretending agents and employees are interchangeable.
4. [**Current capability is enough**](03-current-capability-is-enough.md) — the case does not depend on future model magic.
5. [**The advantage compounds**](04-the-advantage-compounds.md) — the organization improves its system while the underlying tools may improve too.
6. [**The bottleneck moved**](05-the-bottleneck-moved.md) — builder attention, product shaping, review, infrastructure, and release capacity can matter more than raw code output.
7. [**Invert the default**](06-invert-the-default.md) — design around agent execution, then decide where human judgment creates leverage.
8. [**The old process is not enough**](07-the-old-process-is-not-enough.md) — not because it was foolish, but because it was built for different constraints.
9. [**The purpose is outcomes**](08-the-purpose-is-outcomes.md) — delivery actuals and realized customer or business value matter more than preserving familiar rituals.
10. [**Builders, not silos**](09-builders-not-silos.md) — specialties remain; handoffs should not define the operating model.
11. [**The North Star**](10-the-north-star.md) — human-governed, agent-enabled product delivery.
12. [**Company-building implications**](11-company-building-implications.md) — smaller teams, different capital needs, and a different cost structure are downstream effects.
13. [**Beyond product development**](12-beyond-product-development.md) — product delivery is the first proving ground, not the boundary.

## The important caveat

The reader should not come away thinking, “Excellent, lower the bar and let agents ship everything.”

That is not the argument.

Capable agents plus a weak operating model create slop and human review overload. Capable agents plus clear intent, strong environments, gates, evidence, resource visibility, and deliberate builder engagement create leverage.

Resource visibility is not a license for false precision or people surveillance. Initiative tracking is not proof that product value can be cleanly attributed. The discipline is to preserve assumptions, ranges, confidence, actuals, and uncertainty so the next decision is better informed.

The capability is real. The quality of the system around it determines whether that capability becomes an advantage or a mess.
